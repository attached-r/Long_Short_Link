## 调优建议

### 1. 数据库层

- 增加 HikariCP 连接池大小：`maximum-pool-size: 30`
- 优化索引：确保 `short_code` 字段有索引
- 使用数据库连接池监控

### 2. Redis 层

- 增加 Redis 连接池大小
- 使用 Redis 集群或哨兵模式
- 优化缓存策略，提高缓存命中率

### 3. 应用层

- 增加 Tomcat 最大线程数：`server.tomcat.threads.max: 300`
- 调整线程池大小：根据 CPU 核心数调整
- 使用异步处理提高吞吐量

### 4. JVM 层

- 调整堆内存：`-Xms2g -Xmx4g`
- 使用 G1 垃圾收集器：`-XX:+UseG1GC`
- 优化 GC 参数

## 常见问题

### Q1: 压测时大量请求失败

**原因**:
- 数据库连接池耗尽
- 限流策略触发
- 超时时间设置过短

**解决方案**:
- 增加连接池大小
- 调整限流阈值
- 增加超时时间

### Q2: QPS 上不去

**原因**:
- 线程池太小
- 数据库查询慢
- Redis 响应慢

**解决方案**:
- 增加线程池大小
- 优化 SQL 查询
- 检查 Redis 性能

### Q3: 内存溢出

**原因**:
- 堆内存设置太小
- 存在内存泄漏

**解决方案**:
- 增加堆内存：`-Xmx4g`
- 使用内存分析工具排查泄漏

## 监控建议

压测时建议监控以下指标：

1. **应用监控**: CPU、内存、线程池使用情况
2. **数据库监控**: 连接数、慢查询、锁等待
3. **Redis 监控**: 连接数、内存使用、命中率
4. **系统监控**: CPU、内存、磁盘 IO、网络 IO

推荐使用：
- Spring Boot Actuator
- Prometheus + Grafana
- SkyWalking
- Arthas
